# 当直・日直シフト作成 改

このアプリは、公開されているシフト作成アプリをベースにしたカスタム版です。元ページや元リポジトリは変更せず、参照元の `index.html` をコピーして `shift_schedule_custom` 内で独立して動く静的アプリとして改修しています。

## 参照元

- 参照元アプリ: https://norihani.github.io/shift_schedule/
- 参照元GitHub: https://github.com/norihani/shift_schedule

## 起動方法

`index.html` をブラウザで開いてください。外部サーバーやDBは使いません。

ローカルで簡易サーバーを使う場合:

```bash
cd shift_schedule_custom
python3 -m http.server 8000
```

その後、ブラウザで `http://localhost:8000/` を開きます。

## GitHub Pagesで公開する方法

1. `shift_schedule_custom` ディレクトリを公開したいリポジトリへ配置します。
2. GitHubの `Settings` → `Pages` で公開ブランチとフォルダを選びます。
3. `index.html` が公開URLのトップまたはサブディレクトリで開けることを確認します。

## JSON保存/読込

画面右上の「JSONファイル 保存」で現在の設定とシフトをJSONとして保存できます。「JSONファイル 読込」で復元できます。

保存対象には、メンバー、年次、平日回数、休日回数、NG、希望休、前回シフト入力、NG組合せ、休館日、生成済みシフト、警告情報が含まれます。古いJSONで新規項目がない場合は、空データとして補完して読み込みます。

ブラウザ保存は `localStorage` の `shift_schedule_custom_v1` キーを使います。参照元アプリの保存キーとは衝突しません。

## 主要ルール

- 平日は当直のみです。
- 平日回数合計は、平日当直の最小枠から金曜以外の2人目を含む最大枠までの範囲で判定します。
- 休日・祝日・休館日は日直と当直があります。
- 標準モードでは、休日日直は必ず2名、休日当直は1〜2名です。
- 休日回数が標準必要数に足りない場合だけ例外モードになり、休日日直は1〜2名、休日当直は全日1名固定です。
- 平日金曜当直は1年目1人のみです。
- 2年目以上同士は同じ枠に入れません。
- 2年目以上単独は黄色警告です。
- NG日は赤ルールで、生成・交換候補では割り当てません。
- 希望休は黄色警告で、生成・交換候補ではなるべく避けます。
- 当直後は原則中3日、日直後は中2日空けます。
- 交換候補では当直後中3日のみ中2日まで緩和できます。
- 中1日・連日は交換候補にも出しません。
- 休日回数は、休日日直も休日当直も同じ1回として数えます。
- 数字入力では `!` 付きが日直、`!` なしが当直です。

## 数字入力

NG、希望休はメンバーごとに数字入力できます。

例:

```text
4,6!,8,8!
```

- `4`: 4日の当直
- `6!`: 6日の日直
- `8`: 8日の当直
- `8!`: 8日の日直

区切り文字は、半角カンマ、全角カンマ、読点、スペース、改行に対応しています。

前回シフトはNG/希望休表の上にある小さな表で入力します。対象は開始日前3日間のみです。例えば開始日が `2026-10-06` の場合、対象は `10/3`、`10/4`、`10/5` だけです。日直/当直のセルを押すと `○` が付き、今回シフト期間に関わる自動NGへ即反映されます。

期間設定では年セレクトを使えます。年を変更すると、開始日・終了日の月日は維持したまま年だけを変更します。終了月日が開始月日より前の場合は、年またぎ期間として終了年を開始年+1で扱います。

## 既知の注意点

- 生成はブラウザ内のランダム試行方式です。条件が厳しすぎる場合、赤ルール違反なしの完成形が見つからないことがあります。
- 各メンバーの平日回数・休日回数は、ユーザーが変更しない限りアプリ側で自動変更しません。古いデータで未定義の場合のみ2回を補完します。
- 交換候補は探索数と時間に上限があります。候補が多い場合は上位候補のみ表示します。
- 手動編集では赤ルール違反があっても反映できますが、反映後に赤警告として表示します。
- Excel出力は本体機能を優先し、警告一覧は別シートで出力します。
